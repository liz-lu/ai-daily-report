# AI产品岗 · 求职作战台

> 一个面向 27届 AI产品/产品经理求职的自动化工具：每天自动监测各大厂招聘官网是否更新，
> 帮你精准锁定"今天该去看哪家"，并管理投递进度。个人自用、非商业。

**🔗 在线体验**：https://liz-lu.github.io/ai-daily-report/campus-jobs/

---

## 一、我发现了什么真实痛点

秋招/找实习时，我每天要手动刷十几个招聘官网（字节、腾讯、月之暗面、智谱……）看有没有上新岗位。
这件事有三个真实的痛：

1. **重复且低效**：19 个网站挨个点开、翻一遍，大部分时候根本没更新，白刷。
2. **容易漏**：岗位是"什么时候开、什么时候截"没人提醒，错过投递窗口。
3. **投递进度乱**：投了哪些、什么时候投的、进行到哪一步，全靠脑子记，越投越乱。

**核心洞察**：求职者真正的痛点不是"找不到招聘网站"，而是
**"不知道今天哪家动了、值得去看"** + **"自己的投递进度没人帮我管"**。
——这决定了产品不该做"又一个岗位聚合站"，而该做**"变化提醒 + 投递管理"**。

## 二、我怎么设计的、踩了什么坑、怎么兜底

### 方案演进（真实走过的弯路）

| 尝试 | 结果 | 学到的 |
|------|------|--------|
| ① 抓开源秋招汇总仓库(Campus2026) | ❌ 数据停更半年，全是过期岗位 | 依赖第三方数据源 = 把命脉交给别人 |
| ② 直接抓 BOSS/小红书/大厂官网岗位明细 | ❌ 全部需要登录态+JS签名+反爬，服务器脚本抓不到 | 别硬刚反爬，识别技术边界 |
| ③ **改做"官网更新监测"而非"抓明细"** | ✅ 合规、稳定、真正能自动 | **换个问法，把做不到的需求变成做得到的** |

> 第③步是关键的产品决策：**"抓岗位明细"做不到，但"检测页面有没有更新"做得到**——
> 后者同样能解决用户"今天该看哪家"的核心痛点。**绕过技术死结，用可行方案覆盖真需求。**

### 核心机制

```
每天 09:00，GitHub Actions 自动：
①  访问 19 家招聘官网
②  把每个页面正文算成"内容指纹"(MD5)
③  和昨天的指纹比：变了 → 🔴今日有更新；没变 → ○无变化
④  结果写入 monitor.json，前端页面渲染
```

### 踩的坑 & 兜底（做完功能后，我逐个想"哪里会崩"，写测试验证）

| # | 风险 | 后果 | 兜底方案 |
|---|------|------|---------|
| 1 | **假阳性**：官网含 csrf/token/访问量等每次都变的动态内容 | 天天误报🔴，"狼来了"失效 | 指纹算法过滤动态噪音（token/session/长随机串/多位数字），实测：纯动态变化不误报、真上新岗位能检出 |
| 2 | 网站临时抓不到，恢复后 | 误判为"更新" | 抓不到时保留旧指纹、标 unreachable、不判更新 |
| 3 | 某天 19 家"同时更新" | 明显是故障非真更新 | 保险丝：>70% 同变判为批量异常，本轮不采信 |
| 4 | 全网络失败 | 脚本崩溃、任务报红 | 每站独立 try/except，退出码 0，保留旧数据 |
| 5 | **GitHub 60 天无活动自动暂停 workflow** | 自动化悄悄停摆 | 心跳字段(精确到秒)，每次运行输出必不同 → 每天必 commit → 永不触发暂停 |
| 6 | 单个官网改版失效 | — | 单家挂不影响其他家，卡片标 ⚠ 提示 |

> 这一步是我觉得最像"产品经理"的地方：**功能能跑只是起点，
> 真正的工作是穷举失败模式、设计优雅降级**——7 类风险，5 个修复、1 个隔离、1 个可接受。

### 技术选型

- **零成本**：GitHub Actions（免费额度）+ GitHub Pages（免费托管），**无服务器、无 API key**
- **零依赖**：纯 Python 标准库，任何人 Fork 即可跑
- **合规**：只检测公开页面是否变化，不触碰登录态/反爬接口

## 三、数据 / 效果

- **监测覆盖**：19 家（9 家大厂 + 4 家互联网 + 6 家 AI 独角兽），可自行增减
- **首次运行**：18/19 家成功建立指纹基线（修复 URL 编码后 19/19）
- **运行成本**：每次约 30 秒，每月约 15 分钟，远低于 GitHub Actions 免费额度（2000 分钟/月）
- **省下的时间**：把"每天手动刷 19 个网站（约 15 分钟）"压缩成"扫一眼谁亮红灯（约 30 秒）"
- **稳定性**：无人值守自动运行，7 类风险均有兜底

### 三大功能模块

| 模块 | 作用 |
|------|------|
| 📡 官网监测 | 每天标出"今日有更新"的公司，优先去看 |
| 🚪 全部入口 | 19 家官网按类别直达，主动逛也方便 |
| 📌 我的投递 | 记录岗位+追踪状态(想投→已投→笔试→面试→Offer)，自动记录录入/投递时间；数据存本地浏览器 |

## 四、使用与维护

- **在线访问**：https://liz-lu.github.io/ai-daily-report/campus-jobs/
- **加/减监测公司**：编辑 `data/companies.json`（name / category / url / note；URL 中文需百分号编码）
- **手动触发**：GitHub → Actions → "Update Campus Jobs" → Run workflow
- **自动运行**：每天北京时间 09:00（cron `0 1 * * *` UTC）

## 目录结构

```
campus-jobs/
├── index.html              # 前端（监测 / 入口 / 投递 三 Tab）
├── data/
│   ├── companies.json      # 招聘官网入口配置（人工维护）
│   └── monitor.json        # 每日监测结果（自动生成）
└── scripts/
    ├── monitor.py          # 官网更新监测（当前启用）
    └── fetch_jobs.py       # 岗位抓取（备用，社区源活跃时启用）
```

## 说明

- 🔴"今日有更新"表示该官网页面内容变了、值得去看，**不保证一定有你要的岗位**——需点进官网确认。
- 第一次运行只建立基线，**次日起**才开始对比出更新。
- 招聘官网非每日更新，没🔴很正常，有🔴才是信号。
