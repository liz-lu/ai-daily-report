# 如何手动更新（运行 workflow）

> 平时它每天北京时间 **09:00 自动运行**，不用管。
> 只有当你想**立刻拿到最新监测结果**、或怀疑它没跑时，才需要手动运行一次。

---

## 一、手动运行步骤（4 步，约 1 分钟）

### 第 1 步：打开 Actions 页面
浏览器访问：
```
https://github.com/liz-lu/ai-daily-report/actions
```

### 第 2 步：点左侧的 "Update Campus Jobs"
页面**左边**有一列 workflow 名字，点击 **Update Campus Jobs**。
> ⚠️ 必须先点这个名字，右侧才会出现"Run workflow"按钮。

### 第 3 步：点右侧 "Run workflow"
- 页面**右侧**出现灰色的 **`Run workflow`** 按钮，点它
- 弹出小框，分支保持默认 **main**
- 再点小框里绿色的 **`Run workflow`** 确认

### 第 4 步：等它跑完
- 列表最上方出现一条新记录
- 先是黄色圆点 🟡（运行中）→ 约 30秒~1分钟后变 绿色勾 ✅（成功）
- 刷新页面可看到状态变化

---

## 二、怎么确认更新成功了

**方法 A：看 Actions 状态**
运行记录变成 **绿色勾 ✅** = 成功；红色叉 ❌ = 失败（见下方排查）。

**方法 B：看页面"最后检查"时间**
打开 https://liz-lu.github.io/ai-daily-report/campus-jobs/
（打开后按 **Command + Shift + R** 强制刷新清缓存）
顶部"最后检查"时间变成刚才运行的时间 = 更新成功。

---

## 三、常见问题排查

| 现象 | 原因 | 怎么办 |
|------|------|--------|
| 找不到 "Run workflow" 按钮 | 没先点左侧 "Update Campus Jobs" | 先点左侧那个名字，右侧才出现按钮 |
| 运行卡在 "Queued"（排队）很久 | GitHub 服务器排队，正常现象 | 耐心等，通常几分钟内开始；偶尔延迟到 1 小时 |
| 运行变红叉 ❌ | 某步骤报错 | 点进那次运行 → 看红色的步骤 → 截图发我排查 |
| 页面数据没变 | 浏览器缓存 | 按 **Command + Shift + R** 强制刷新 |
| 页面全显示"监测中" | 第一次运行只建基线 | 正常，次日起才对比出更新 |
| 全部公司显示 unreachable | 那次网络异常 | 再手动 Run 一次即可；单次失败不影响数据 |

## 四、关于自动运行

- **频率**：每天北京时间 09:00（配置在 `.github/workflows/update-jobs.yml`，cron `0 1 * * *` 是 UTC 时间）
- **会不会自己停**：不会。脚本每次运行都会产生一次提交（心跳机制），
  所以永远不会触发 GitHub"仓库 60 天无活动自动暂停 workflow"的规则。
- **成本**：每次约 30 秒，每月约 15 分钟，远低于 GitHub Actions 免费额度（2000 分钟/月）。

## 五、想改监测哪些公司

编辑 `campus-jobs/data/companies.json`，照格式增删：
```json
{"name": "公司名", "category": "大厂", "url": "招聘官网URL", "note": "备注"}
```
> URL 里如果有中文，要用百分号编码（如 `产品` → `%E4%BA%A7%E5%93%81`），否则监测会失败。

改完 commit，下次运行就生效。
